  1. Opening the binary in a debugger and start setting breakpoints to figure out where the key is used and how is it used.
  2. Use objdump -d to get a assembler dump of the binary and to static analysis

In any case, when you find the relevant part of the program you will have to patch the binary. Actually finding the password is a bit tricky…
