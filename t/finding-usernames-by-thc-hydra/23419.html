Hey @Jason,

The way that Hydra knows that a particular credential works is using the ‚ÄúFail String‚Äù which in your case is set to ‚Äúincorrect‚Äù.

'/phpmyadmin/index.php:pma_username=^USER^&pma_password=^PASS^:--->incorrect<---'

So, after trying out a username/password combo, if the response doesn‚Äôt contain the word ‚Äúincorrect‚Äù, then hydra considers that username/password combo to be correct.

It is possible that the ‚ÄúLogin Failed‚Äù page doesn‚Äôt contain the ‚ÄúFail String‚Äù which you‚Äôve provided and hydra eventually thinks that authentication was successful for all combinations.

To Fix,

  1. Check if the login failed page has this keyword
  2. Add the -f flag so that hydra will terminate on the first successful attempt.
