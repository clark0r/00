prometheus (prometheus) November 13, 2022, 10:00pm 1

I know there are algorithms like gzip to compress files, but I am wondering if there are alternatives to compressing files for my use case. I have created a P2P browser network that stores data in LocalStorage (which has a limit of 5mb of storage). Here is the process:

  • Turn file into an ArrayBuffer
  • Split bytes into chunks of 50kb
  • Upload chunks to the network (peers get the data and store it in their LocalStorage)

The problem is that I had to increase the quota/limit for LocalStorage for file uploads to even work.

Currently I am thinking of an algorithm that looks at the byte array and then examines the array by chunks. Assume the array is of length N, the algo would start off by examining chunks with length N, then N-1, then N-2 and so on. If there was a large chunk of 0s of length L, for example, it would store this chunk as 0xL instead of the L number of 0s. If there was a chunk of 1s of length 43, for example, it would do the same, making it 1x43. Putting these compressed chunks in order would allow for lots of compression with varied chunk sizes.
