Basic Base 10 Counting

First I‚Äôm going to talk about how you learned to count. I know this is obvious, but stick with me. You probably learned to start counting at one, but that‚Äôs not the correct way to count as we programmers know ;). To count, you just go to the next digit. 0, 1, 2, 3, 4, ‚Ä¶, 9. Once you get to nine, there is no next digit, you have to start using two digits to represent your number. So, add a new digit to the left of your number (it actually has always been there, it was just zero), set it to one, and reset all digits to the right back to zero. Now continue incrementing the right-most digit.

Base Explanation

Every number system has a base aka radix. Decimal, the number system we have used since childhood, is base 10. This means it has 10 possible single digits (or 10 different characters if you want to think of it that way). The largest single digit in a number system will always be one less than the radix since zero is one of those possible single digits. Binary is base 2, meaning it only has two possible single digits: 0 & 1.

Counting in other number systems

Counting in other number systems works the same exact way as base 10.

Let‚Äôs count in binary (base 2):

  • Start at zero
  • 0
  • Increment by one
  • 0 + 1 = 1
  • Increment by one again. But wait, base two has no single digit for 2. That means we need to add a new digit and set the right digit back to zero
  • 1 + 1 = 10
  • Increment by one again
  • 10 + 1 = 11
  • Increment by one. Same issue as last time, just happens twice
  • 11 + 1 = 100
  • If this doesn‚Äôt make sense to you: all the digits are at their maximum value, so we need a new digit and we reset all the digits before it back to zero.

Hopefully that made sense to you and you were able to count even higher on your own.

Let‚Äôs try out octal (base 8) now:

  • Start at zero
  • 0
  • Increment by one
  • 1
  • This is going to take a while, let‚Äôs fast forward to seven
  • 7
  • Increment by one. Since this is base 8, seven is the largest single digit we have, so we need a new digit
  • 10
  • Let‚Äôs fast forward again
  • 17
  • Here is a case we haven‚Äôt seen yet. You might be tempted to go to 100, but that‚Äôs wrong. We just increment the second digit and reset the digit to the right

  • 1300. You increment the third number and reset the two numbers to its right

Conclusion

Binary, Octal, Decimal, Hex, etc all work the same. Once you understand the similarities, you can understand any number system.
