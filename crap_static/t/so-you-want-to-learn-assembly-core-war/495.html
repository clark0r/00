Core War

Core War is a 1984 programming game created by D. G. Jones and A. K. Dewdney in which two or more battle programs (called "warriors") compete for control of a virtual computer. These battle programs are written in an abstract assembly language called Redcode. At the beginning of a game, each battle program is loaded into memory at a random location, after which each program executes one instruction in turn. The goal of the game is to cause the processes of opposing programs to terminate (which h...

Despite of being a fun game, it is a nice way to start with low level programming. The machine does not have registers or stack, just memory and some flags. The number of instructions is small (around 16, depending on the version you try) and only a couple of memory addressing modes are allowed. There are some kind of threading also.

Take a look to the wiki page (specially the part about the use of genetic algorithms to generate code )
