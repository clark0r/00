The ‚Äòhack‚Äô

The hack in this case is just trying to make sense of the encrypted message and figuring out how this algorithm works by testing the script itself which we have access to.

What now?

It‚Äôs time to recall basic character encoding techniques!
As we can look up at the link above a small ‚Äòd‚Äô corresponds to ‚Äò100‚Äô in ASCII!

We found out how a single character without choosing a password is encrypted!

But how does the password field make any sense now?

Let‚Äôs try encrypting our single ‚Äòd‚Äô with the password: d

And with some mad black magic skills we can see that our single encrypted ‚Äòd‚Äô still has the form of .XX.XX.XX, but this time the numbers add up to 200!

It‚Äôs spoiler time :
This behavior continues. Every character you put in the password field gets converted to its ASCII value and is added to each ASCII value from the characters of the clear text.
(the longer the password the higher the numbers in the end.)

So now it‚Äôs time to decrypt our message.
With this knowledge on board we‚Äôre almost ready to do this!

How do we get the password for decrypting??

This can be done through a frequency analysis.
We add up every ‚Äò3 chunk‚Äô and look at which element appears the most often.
Now some of you may think: ‚Äúhey that‚Äôs simple, it‚Äôs English language so it‚Äôll be the ‚Äòe‚Äô.‚Äù

  • Secondly: In general, when writing a message one uses more ‚Äòspace characters‚Äô than ‚Äòe‚Äô characters.

‚Üí space corresponds to the numerical value of 32 in ASCII

And that‚Äôs it. Subtracting the ASCII value for space from the added chunk with the most occurrences in the cipher results in the password key!
Now as a last step we can subtract this password key from every added chunk from before to get the clear text message.
